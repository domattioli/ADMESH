# PFIX Bit-Exact Preservation Contract

**Issue**: #46  
**Status**: Documentation  
**Date**: 2026-05-03  

## Overview

The `pfix` (fixed points) parameter in ADMESH provides a guarantee: **all points in `pfix` will appear in the final mesh with bit-exact coordinates**.

This contract is critical for coastal modeling workflows where certain nodes must be preserved (e.g., inlet positions, weir locations, structure foundations).

## API Contract

### In `Domain`

```python
pfix: np.ndarray | None = None
```

**Shape**: `(K, 2)` where K = number of fixed points  
**Type**: Float array (same precision as mesh nodes)  
**Guarantee**: Every point `pfix[i]` will appear as a mesh node with identical coordinates.

### In `triangulate()`

```python
mesh = admesh.triangulate(
    domain,
    h0=0.1,
    # ... other params
)
```

Fixed points are supplied via `domain.pfix`. No separate `pfix` parameter in the public API.

## Behavior

### What `pfix` Does

1. **Preserves coordinates**: Every point in `pfix` appears in `mesh.nodes` with unchanged `(x, y)` values.
2. **Respects ordering**: Points maintain their row order in the output (though mesh node numbering may differ).
3. **Scales with mesh**: Fixed points participate in sizing (`h_min`, `h_max` applied around them) but are never moved.

### What `pfix` Does NOT Guarantee

- **Element quality**: Triangles incident to fixed points may have lower quality (e.g., q < 0.3) if fixed points are too close together.
- **Mesh connectivity**: The elements around fixed points are generated by distmesh; their topology is not constrained.
- **Bathymetry**: Fixed point (x, y) coordinates are preserved, but any per-node bathymetry values must be supplied separately in `mesh.bathymetry`.

## Numerical Guarantees

- **Precision**: Fixed point coordinates are stored as `float64` (IEEE 754 double precision).
- **Bit-exact match**: Coordinates are preserved **exactly** (no rounding, no quantization) through the entire meshing pipeline.
- **Test**: Reference test `test_pfix_bit_exact_preservation()` in `tests/test_api.py` verifies this guarantee on every release.

## Implementation Details

### Flow

1. User provides `domain.pfix` as `(K, 2)` array.
2. `triangulate()` passes `pfix` to `admesh.distmesh.distmesh2d(..., pfix=pfix)`.
3. Distmesh's force-balance solver **prevents** fixed points from moving:
   - Fixed points do NOT receive force updates.
   - They act as "anchors" to constrain the surrounding mesh.
4. Output mesh contains all `pfix` coordinates unchanged in `mesh.nodes`.

### Key Code Paths

- **API entry**: `admesh/api.py::triangulate()` → extracts `pfix` from domain
- **Distmesh driver**: `admesh/distmesh.py::distmesh2d(pfix=...)` → passes to force solver
- **Force solver**: `admesh/routine.py::ADmeshRoutine.run()` → fixes point rows in Jacobian
- **Output**: `routine.py` → returns mesh with pfix coordinates untouched

## Testing

### Unit Test: `test_pfix_bit_exact_preservation()`

Location: `tests/test_api.py`

Verifies:

1. **Count**: Output mesh has ≥ K nodes (fixed points + generated nodes).
2. **Membership**: Each `pfix[i]` appears in `mesh.nodes` exactly once.
3. **Bit-exact match**: `np.allclose(found_pfix_rows, pfix, rtol=0, atol=0)`.
4. **No ordering assumption**: Fixed points may appear in any order in the output.

```python
def test_pfix_bit_exact_preservation():
    domain = Domain(...)
    domain.pfix = np.array([[0.5, 0.5], [0.0, 1.0]])
    mesh = admesh.triangulate(domain)
    
    # Verify membership (order-independent)
    for pfix_point in domain.pfix:
        found = np.any(
            np.allclose(mesh.nodes, pfix_point[None, :], rtol=0, atol=0),
            axis=1
        )
        assert np.any(found), f"pfix point {pfix_point} not in mesh!"
```

### Real-World Test: Fort.14 with Weir Nodes

Location: `tests/test_fort14_paired.py` (forthcoming)

Verifies that boundary node coordinates (including weir attachment points) round-trip through `Domain.from_mesh()` → `triangulate()` with bit-exact preservation.

## Example

### Inlet Preservation

```python
import admesh
import numpy as np

# Domain with an inlet at (1.0, 0.5)
domain = Domain(...)
domain.pfix = np.array([[1.0, 0.5]])  # Inlet point must be in mesh

mesh = admesh.triangulate(domain, h0=0.1)

# Verify inlet is preserved
assert np.any(
    np.allclose(mesh.nodes, [1.0, 0.5], rtol=0, atol=0),
    axis=1
), "Inlet point lost!"
```

## Historical Context

### Why This Contract Matters

- **ADCIRC coastal models**: Weir locations, structure positions, and inlet boundaries must remain fixed across remeshing.
- **Bathymetry depth**: Depth at fixed points is known and cannot be re-interpolated without error.
- **Physical validity**: Moving a fixed point even 1 mm may invalidate real-world coastal geometry.

### Evolution

- **v0.1**: pfix supported but not formally documented.
- **v0.2+**: Contract formalized and tested explicitly.
