# Implementation Plan: Fix Domain.from_mesh Ring Sorting

**Branch**: `claude/pensive-goodall-r2Td2` | **Date**: 2026-04-26 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/003-fix-outer-ring-sorting/spec.md`

## Summary

**Primary Requirement**: When recovering a multiply-connected domain from a real-world mesh via `Domain.from_mesh(src)`, the function must correctly identify the outer boundary ring (by **signed area**, not node count), so that downstream `triangulate(domain, ...)` operations produce valid meshes with correct spatial extent.

**Technical Approach**: 
1. Implement a `_ring_area(ring_segment, nodes)` helper using the 2D shoelace formula to compute signed area.
2. Replace the `rings.sort(key=len, reverse=True)` criterion in `_derive_boundary_segments` with area-based sorting.
3. Update `Domain.from_mesh` to use the sorted rings correctly (outer = index 0, holes = index 1+).
4. Add unit test covering the multiply-connected fixture (wnat_test.14) where interior rings have more nodes than the outer ring.
